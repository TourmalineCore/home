# The Process of Layout and Styling via TDD

## Status
Accepted

## Context

It is necessary to test components in isolation, but since there are difficulties with component tests, we use a workaround by testing via e2e, but accessing the component on a separate page where it will also be isolated.

Testable components must be added [here](https://github.com/TourmalineCore/pelican-ui/blob/master/src/pages/components/%5B%5B...component%5D%5D.tsx)

## Decision
1. Create an E2E test in the component's folder (e.g., ```src/components/home-page/Hero/Hero.spec.tsx```), in which we navigate to the page where the target component is rendered.
2. Set breakpoints — width (by default it corresponds to the MOBILE breakpoint).
3. Run the test. On the first run, a folder for screenshots will be created (```playwright-tests\screenshots\base\src\components```), where snapshots of the component's current state are generated.
For Playwright to locate the component, we use the **data-testid** attribute on the component. If there is no tag with data-testid yet, Playwright will not be able to find anything and will not create the folder.
4. Export images in PNG format from Figma, replace the images in the screenshots folder with them. It is important that the filenames are exactly the same as those generated by Playwright.
5. Develop the component, comparing it with the reference image (using the tools in the Playwright UI: difference, actual, expected, side by side, slider).
6. When the maximum acceptable similarity is achieved, replace the reference images from Figma with snapshots of the component's actual current state. To do this, you can clear the folder containing the PNG files from Figma and rerun the test. You can also run the command ```npx playwright test --update-snapshots```, which runs all tests and updates the mismatched images.

### Notes
1. It is necessary to work on the project through the devcontainer so that the reference snapshots are identical to those in the pipeline, regardless of your operating system.
2. A specific naming convention for images from Figma.<br/>
	Example:<br/>
		**header-desktop-chromium-linux**<br/>
		**header-desktop-firefox-linux**
